Linters e Formatters

Um pouco mais sobre essas ferramentas

Linters

O que é e onde surgiu

1978

Stephen C. Johnson

Nessa época o Stephen estava interessado em como criar um operador 'or' exclusivo para a linguagem B

com crescente dificuldade, ele elaborou o chamado Lint, e o que ele faz?

Esta ferramenta é capaz de dedurar possiveis falhas no seu código

Ela executa o seu código em partes, sendo capaz de verificar váriaveis fora de escopo, erros de digitação e na época era capas de apontar falhas no uso de memória e otimização, o que hoje é dever do copilador (para facilitar a vida do desenvolvedor).

De forma simples

O Lint era uma ferramenta capaz de fazer uma varredura no seu código antes da copilação, assim podendo te direcionar a possíveis falhas e também te da liberdade de definir as regras obedecidas na sua execução.

Assim surgiram os programas chamados

Linters

Linters

Na atualidade

Hoje um linter acumulou muito mais funções além de análise do código de forma estrutural,

ele já é capaz de indicar problemas de segurança, vazamento de memória e pode padronizar o código dentro de uma especificações pré-definidas.

É essencial

O uso de linters hoje é basicamente algo padrão, pois evita muito erros e é responsável por manter um ambiente limpo, seguro e propício a manutenção.

Formatters

O braço direito de um Linter

Formatters é legibilidade

Um formatter ou formatador é a ferramenta capaz de corrigir erros encontrados na escrita do código e aplicar correções

fugindo da verificação de lógica, ele irá obedecer regras de escrita como ponto e virgula, identação, e até utilização de 'camelCase' e como seu usuário prefirir.

Manutenção

hoje todos os linters também são formatters

a formatação do código é a maior responsável por uma manutenção de vários desenvolvedores num mesmo projeto. Com o passar do tempo e aumento de complexidade, a legibilidade do código é o principal fator para atualização e permanencia de seu funcionamento.

Na pratica

Uso do ESLint

ESLint

É a ferramenta Linter mais utilizada atualmente

entendendo a importância do uso de linters, agora vamos fazer a instalação desta ferramenta no nosso projeto inicial sem o create-react-app.

Utilizando o terminal e o npm vamos do inicio


              $ mkdir projeto-teste123 //criando o diretorio
              $ cd projeto-teste123 //entrando na pasta
              $ touch index.html //o arquivo do projeto
              $ npm i -y //iniciando o npm que vai gerenciar as ferramentas
              $ npm install -d eslint //instalar o eslint no projeto '-d' apenas para desenvolvimento
              $ npm install -d prettier //instalar o prettier pra usar junto
              $ code .  //vai abrir a pasta no vscode

Algumas regras

No arquivo .eslintrc.json


              "rules": {
                "indent": [
                    "error",
                    "tab"
                ],
                "linebreak-style": [
                    "error",
                    "unix"
                ],
                "quotes": [
                    "error",
                    "single"
                ],
                "semi": [
                    "error",
                    "always"
                ],
                "eqeqeq": 0, // 0="off" 1="warn" 2="error"
                //essa regra aparece no uso de '==' ao invez de '==='
                "no-magic-numbers":0, // evita o aparecimento de numeros nao declarados
                no-unused-vars:0, // O mais comum, que evita variaveis nao utilizadas
            }
